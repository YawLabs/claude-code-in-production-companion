# Ops

Operational guides for running the course as a product -- LemonSqueezy setup, fulfillment automation, refund policy, support queue triage. Not part of the course content; published here for transparency.

```
ops/
+-- lemonsqueezy-setup.md       # how the storefront is configured
+-- fulfillment.md              # how purchases trigger access
+-- support-triage.md           # how incoming questions get routed
```

If you're building an adjacent course-as-a-product setup, these are the patterns I converged on. The shape is shared with the *Production MCP Servers* course (`yawlabs/production-mcp`) -- both courses use the same fulfillment pipeline.
